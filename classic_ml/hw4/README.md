# hw_boosting

## Как сдавать решение и в какой срок

### Оценка
За задание можно получить 10 баллов, если сдать до Soft Deadline. После него, но до Hard Deadline -- 5 баллов. После решение не оценивается.

### Дедлайны
**Soft Deadline - 30 декабря в 23:59.**

**Hard Deadline - 12 января в 23:59.**

### Формат сдачи
По аналогии с первым дз:
1. Нужно завести ветку `hw4` и вести свою разработку там.
2. Свою реализацию пропишите в файле `hw4/solution.py`, для вас уже готова заготовка класса.
3. Локально тесты можете запускать так:
    ```
   python -m pytest --log-cli-level INFO hw4/tests/
   ```
4. Как только считаете, что решение готово, заводите Merge Request из ветки `hw4` в `master`.
5. Если с прошлым МР-ом что-то было не так и вы решили завести новый, закройте, пожалуйста, старый самостоятельно. Это делается на странице МР-а Троеточие -> Close merge request.
6. Если пайплайн полностью проходит (все зеленное), бот отписал комментарий с баллами в ваш MR, ставьте в assignee `Иван Борисов`.
7. Profit!

### Что нужно сделать
- В solution.py находится заготовка класса GBCustomClassifier.
- Нужно имплементировать нужные методы в заготовке класса. Можно добавлять новые методы
- Нужно поддержать параметры, описанные в сигнатуре.
- Логику алгоритма можно реализовать по разному, но если что, достаточно классического ванильного градиентного бустинга с лекции, без XGBoost. По качеству запас есть.
- GBCustomClassifier должен поддерживать мультиклассовую классификацию.
- В реализации решения можно пользоваться только DecisionTreeRegressor из sklearn, numpy и стандартными библиотеками Python.
